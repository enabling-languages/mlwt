mlwt: Language specific support in LCG fonts

CSS classes for use with LCG fonts developed by SIL International.

Language specific features

The Opentype tables provide the locl feature for Serbian and Vietnamese. The best way to use the Serbian and Vietnamese variant glyphs is by applying correct language tagging in your HTML document.

Graphite fonts provide additional support not available via locl OpenType features.

Graphite language specific support for Teribe (Naso) and Konai use the Graphite brig feature, there is no equivalent Opentype feature and there is no locl support for these languages.

Graphite language specific support for Ozumacín Chinantec uses the Graphite chtn feature, which is equivalent to teh OpenType cv90 feature. There is no locl support for Ozumacín Chinantec. A langauge specifc rule has been added to provide langaueg speciifc support for both Graphite and OpenType rendering.

Vietnamese-style diacritics

Use language tagging

The Graphite and OpenType rules provide language specific features for Vietnamese. It is not necessarily to directly specify the graphite or opentype features for Vietnamese. The features will be activated automatically via language tagging.

To enable Vietnamese rendering add lang="vi" to the appropriate elements.

Some Latin script minority languages in Vietnam may also require Vietnamese typographic preferences. In this case either directly access the required graphite or opentype feature or write a CSS rule using a language pseudo-selector.

Serbian-style italic alternates

Use language tagging

The Graphite and Opentype rules provide language specific features for Serbian. It is not necessarily to directly specify the graphite feature for Serbian. The features will be activated automatically via language tagging. There is no direct opentype cvNN or ssNN feature for accessing Serbian italic variants.

To enable Serbian rendering add lang="sr" to the appropriate elements.

N.B. There is currently no support for Macedonian italic variants. When using Graphite rendering it is possible to access the serb feature.

Support for other languages in mlwt

Yolŋu languages

mlwt.lcg.css contains the following rule for Yolŋu languages:


:lang(dhg), :lang(jay), :lang(lja), :lang(dwu),
:lang(djr), :lang(gnn), :lang(guf), :lang(dax),
:lang(dwy), :lang(rit), :lang(dji), :lang(djb) {
    font-feature-settings: "Engs" 2, "cv43" 2;
}

This rule allows Yolŋu text to be displayed with the correct version of U+014A (Ŋ).

Ozumacín Chinantec


:lang(chz) {
    font-feature-settings: "chtn" 1, "cv90" 1;
}

Moro


:lang(mor) {
    font-feature-settings: "bowl" 1, "ss04" 1, "R_tl" 1, "cv55" 1;
}

This rule allows Moro text to be displayed with alternative glyphs for U+0111 (đ) and U+2C64 (Ɽ).

Sami languages

Write your own language rules

It is possible to create your ownrules to tailor the typographic display for the languages you need to support:

  1. Identify the CSS selector you wish to apply the rules to. If you choose to use a language psuedo-selector, select the most appropriate BCP-47 language tag
  2. Identify the graphite and opentype features you require
  3. When writing a rule using font-feature-settings inclcude the graphite feature before the opentype feature, if the features have different labels.
